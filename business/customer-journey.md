## customer-journey.md  

### Product: **med‑bridge** – Secure remote collaboration & medical‑data transfer platform  

| Phase | Trigger Event | Primary Friction Points | User Emotions | Opportunities to Delight | Success Metric |
|-------|---------------|--------------------------|---------------|--------------------------|----------------|
| **Aware** | • A clinician reads about a data‑breach at a peer hospital.<br>• Hospital IT director receives a compliance audit notice (HIPAA, GDPR, etc.).<br>• Conference session on “Tele‑health security”. | • Over‑load of generic tele‑health solutions.<br>• Unclear regulatory differences across regions.<br>• Skepticism about “another SaaS” adding complexity. | Curious, cautious, slightly anxious about compliance risk. | • Publish a concise, data‑driven whitepaper (“Secure Tele‑health Collaboration: 5 % breach reduction in 6 months”).<br>• Offer a short explainer video that shows a single‑click encrypted file share between doctor and patient.<br>• Provide a compliance checklist download that maps med‑bridge features to HIPAA, GDPR, Thai PDPA. | **Impression rate** – % of target clinicians who view the whitepaper/video (goal ≥ 12 %). |
| **Consider** | • IT team evaluates vendors after the audit.<br>• Clinician asks for a tool that lets them review imaging results with a specialist in another hospital. | • Complex pricing tables.<br>• Limited case studies in the healthcare niche.<br>• Concerns about integration with existing EHR/EMR systems. | Analytical, hesitant, risk‑averse. | • Interactive ROI calculator (e.g., “Save $X per year by reducing re‑scans”).<br>• Publish a “Healthcare‑specific” case study (e.g., remote stroke‑team collaboration in Thailand).<br>• Offer a sandbox demo with pre‑loaded DICOM data that can be accessed via a secure link in 2 minutes. | **Qualified‑lead conversion** – % of aware prospects that request a sandbox demo (goal ≥ 8 %). |
| **Try** | • Decision‑maker signs up for a 14‑day pilot (no credit card).<br>• Clinician receives an invitation to join a test group for remote case reviews. | • Onboarding time (account creation, device enrollment).<br>• Learning curve for secure sharing workflow.<br>• Need for quick support if a patient’s data fails to upload. | Hopeful, focused, occasionally frustrated by any delay. | • One‑click SSO provisioning linked to hospital’s IdP (Okta, Azure AD).<br>• Guided “first‑share” wizard with real‑time progress bar and success toast.<br>• Dedicated 24‑h “pilot champion” (chat/phone) to resolve issues instantly. | **Pilot activation rate** – % of signed‑up pilots that complete at least 3 secure transfers within 7 days (goal ≥ 70 %). |
| **Adopt** | • Pilot metrics show ≥ 30 % reduction in data‑transfer time and zero compliance incidents.<br>• Finance approves annual contract. | • Contract negotiation (terms, data‑ residency).<br>• Migration of legacy files into med‑bridge.<br>• Scaling user licenses across departments. | Confident, satisfied, but vigilant about SLA adherence. | • Provide a “migration concierge” service (automated bulk upload + validation).<br>• Offer a custom SLA dashboard (uptime, encryption status, audit logs).<br>• Early‑access to new features (e.g., AI‑assisted image annotation) for adopters. | **Closed‑won rate** – % of pilots that convert to paid contracts (goal ≥ 45 %). |
| **Expand** | • Hospital adds new specialties (e.g., oncology) to the platform.<br>• Regional health network requests cross‑institution collaboration. | • Need for role‑based access controls across multiple orgs.<br>• Pricing model for multi‑site usage.<br>• Training for new user groups (nurses, admin staff). | Empowered, eager to innovate, looking for ROI amplification. | • Introduce “Enterprise Collaboration Hub” – a shared workspace with granular permissions.<br>• Volume‑based discount tiers and referral credits for partner hospitals.<br>• Quarterly health‑check webinars + certification badge for staff. | **Expansion revenue** – % increase in ARR from existing customers within 12 months (goal ≥ 25 %). |

### Visual Summary (textual)

```
Aware  →  Consider  →  Try  →  Adopt  →  Expand
  |          |          |        |          |
  ▼          ▼          ▼        ▼          ▼
Trigger → Pain → Pilot → Contract → Upsell
```

Each phase is designed to **reduce friction**, **build trust**, and **quantify value** so that med‑bridge moves from a vague awareness of security risk to a strategic, revenue‑generating partnership across healthcare networks.
